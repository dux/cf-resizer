# cf-resizer - AI Agent Instructions

Cloudflare image CDN toolkit. Stores originals in R2 (content-addressed by SHA1), serves resized/converted images via CF Image Resizing, caches at CDN edge + R2 + browser.

## Project layout

```
src/index.ts          Cloudflare Worker (Hono app) - the deployed service
index.ts              NPM package entry, re-exports cfResizeUrl
lib/env.ts            Config loader for CLI tools (reads .cf-resizer.yaml)
lib/cf-resizer.ts     TypeScript client library (URL builder)
lib/cf-resizer.rb     Ruby client library (URL builder)
bin/cf-resizer        CLI dispatcher (routes subcommands to bin/ scripts)
bin/cf-install        Interactive setup wizard (DNS, R2 bucket, deploy)
bin/cf-prepare        Local image resize + webp conversion via sharp
bin/cf-upload         Auto-prepare + upload to worker
bin/cf-list           List images via worker API
bin/cf-delete         Delete images via worker API
bin/cf-asset          Upload arbitrary files to /assets/
bin/cf-test           Bash test suite (4 phases)
test/fixtures/        Test images (generated by test/create-fixtures.sh)
doc/plan.md           Architecture and implementation plan
.cf-resizer.yaml      Local config (gitignored) - domain, secret, bucket, max_size
```

## Stack

- **Language:** TypeScript (ESM, `"type": "module"`)
- **Runtime:** Bun (CLI scripts), Cloudflare Workers (deployed worker)
- **Framework:** Hono v4 (web framework for the worker)
- **Image processing:** sharp (CLI local resize), CF Image Resizing (worker on-demand transforms)
- **Storage:** Cloudflare R2 (native binding, not AWS SDK)
- **Bundler/deployer:** Wrangler v4 (bundles TS directly, no build step)
- **Package manager:** Bun (`bun.lock`)
- **Types:** `@cloudflare/workers-types` (R2Bucket, ExecutionContext, ScheduledEvent)

## Key concepts

- **Content-addressed storage:** Images stored at `raw/{sha1}` in R2. Same bytes = same URL. SHA1 computed from raw image bytes via `crypto.subtle.digest`.
- **Transform cache:** Transformed images stored at `cache/{sha1}_w{N}_h{N}_cover_q{N}_{format}` keys in R2.
- **Three-layer caching:** CDN edge (`caches.default`) -> R2 transform cache -> browser (`Cache-Control: public, max-age=31536000, immutable`).
- **Format negotiation:** SVG served as-is. Otherwise: Accept `image/avif` -> avif, Accept `image/webp` -> webp, fallback -> jpeg.
- **Auth:** Admin endpoints use secret in URL path (`/upload/{secret}`), no headers. `checkAuth()` compares `c.req.param("secret")` to `env.ADMIN_SECRET`.
- **Config discovery:** `.cf-resizer.yaml` found by walking up from cwd (like `.gitignore`). Values support `$ENV_VAR` expansion.
- **wrangler.toml is gitignored** - generated dynamically by `bin/cf-install` into OS temp directory.

## Worker routes (src/index.ts)

| Method | Route | Purpose |
|--------|-------|---------|
| GET | `/:sha1` | Serve original (format negotiation) |
| GET | `/r/:transforms/:sha1` | Path-based resize/crop (`/r/w300h200q75/{sha1}`) |
| GET | `/r/:sha1?w=&h=&c=&q=` | Query-based resize/crop |
| GET | `/origin/:sha1` | Internal origin for CF Image Resizing fetch |
| PUT | `/upload/:secret` | Upload image (body = raw bytes) |
| GET | `/list/:secret?limit=N` | List images (default 100, max 1000) |
| DELETE | `/delete/:secret/:sha1` | Delete image |
| PUT | `/assets/:secret/:path` | Upload asset file |
| GET | `/assets/:path` | Serve asset (public, cached) |
| GET | `/favicon.ico` | Orange square SVG |
| GET | `/` | Text banner |
| GET | `/debug` | JSON with cache status |

Transform params: `w` = width, `h` = height, `c` = cover crop (square or WxH), `q` = quality 1-100. All dimensions clamped to MAX_SIZE (default 1600).

## Code conventions

- All bin/ scripts use `#!/usr/bin/env bun` shebang (except cf-test which is bash)
- No unit test framework (vitest/jest) - tests are in bash (`bin/cf-test`)
- Minimal YAML parser in lib/env.ts (no yaml dependency) - regex-based, handles comments and quoted values
- MIME detection via magic bytes (`detectMimeFromBytes` in src/index.ts), not file extensions
- Error responses are plain text with appropriate HTTP status codes
- `c.executionCtx.waitUntil()` for background R2 cache writes
- R2 keys use prefixes: `raw/` for originals, `cache/` for transforms, `assets/` for static files
- Cron trigger at `0 3 * * *` runs `cleanCache()` - deletes cache/ entries older than 30 days

## When modifying the worker (src/index.ts)

- The Hono app type is `Hono<{ Bindings: Bindings }>` where Bindings has `BUCKET: R2Bucket`, `ADMIN_SECRET`, `DOMAIN`, `MAX_SIZE`
- CF Image Resizing is invoked via `fetch(originUrl, { cf: { image: imageOpts } })` - the worker fetches from itself at `/origin/:sha1`
- All image responses must include `Cache-Control: public, max-age=31536000, immutable`
- The CDN cache layer in the default export wraps `app.fetch()` and adds `X-Cache: HIT/MISS` header
- Admin endpoints skip CDN cache (see the `skip` check in the fetch handler)
- SHA1 params are validated with regex `[0-9a-f]{40}` in route patterns

## When modifying CLI tools (bin/)

- All bun scripts import config from `../lib/env.ts` which exports `domain`, `secret`, `bucket`, `maxSize`, `baseUrl`
- `lib/env.ts` exits the process if `.cf-resizer.yaml` is not found or missing required fields
- cf-prepare handles SVG as passthrough (copy unchanged), all other formats -> resize to max_size + webp q90
- cf-upload auto-prepares locally (sharp resize + webp) then PUTs to worker. SVG uploaded as-is.
- cf-install generates wrangler.toml in `tmpdir()` and uses `flarectl` for DNS and `wrangler` for R2/deploy

## When modifying client libraries (lib/)

- `lib/cf-resizer.ts` exports `cfResizeUrl(sha1, opts?)` - builds URL with optional transforms
- `lib/cf-resizer.rb` exports `CfResizer.url(sha1, opts)` and `cf_resize_url` convenience method
- Both discover domain from `.cf-resizer.yaml` by walking up from cwd, cached in module scope
- Both support `domain:` override in opts
- Transform order in URL: crop first, then width, height, quality (e.g. `/r/c200w300q75/{sha1}`)
- `index.ts` at project root re-exports `cfResizeUrl` for `import { cfResizeUrl } from "cf-resizer"`
